# API & Database

---

## Supabase Project

Route Book uses Supabase as its backend — PostgreSQL for data storage and GoTrue for authentication.

---

## Database Schema

### `travel_records`

The only application table. Stores one row per trip entry.

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| `id` | `uuid` | No | Primary key, generated by the client |
| `user_id` | `uuid` | No | Foreign key → `auth.users.id` |
| `date` | `date` | No | Trip start date |
| `end_date` | `date` | Yes | Trip end date (null for same-day entries) |
| `from` | `text` | No | Origin stored as `"City \| Country"` |
| `to` | `text` | No | Destination stored as `"City \| Country"` |
| `country` | `text` | Yes | Legacy country field — used as fallback |
| `purpose` | `text` | Yes | City/location label |
| `notes` | `text` | Yes | Free-form notes |
| `created_at` | `timestamptz` | No | Auto-set by Supabase |

## Row-Level Security (RLS)

RLS must be enabled on `travel_records`. Without it, any authenticated user can read or modify any other user's data.

### Required policies

**Select — users can only read their own records:**
```sql
CREATE POLICY "select_own_records"
  ON travel_records
  FOR SELECT
  USING (auth.uid() = user_id);
```

**Insert — users can only insert records for themselves:**
```sql
CREATE POLICY "insert_own_records"
  ON travel_records
  FOR INSERT
  WITH CHECK (auth.uid() = user_id);
```

**Update — users can only update their own records:**
```sql
CREATE POLICY "update_own_records"
  ON travel_records
  FOR UPDATE
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);
```

**Delete — users can only delete their own records:**
```sql
CREATE POLICY "delete_own_records"
  ON travel_records
  FOR DELETE
  USING (auth.uid() = user_id);
```

### Verifying RLS is on

In Supabase Dashboard: **Table Editor → travel_records → RLS** — confirm the toggle is enabled and all four policies are present.

## Authentication

Authentication is handled entirely by Supabase Auth (GoTrue). The app uses:

| Feature | Supabase method |
|---------|----------------|
| Sign up | `supabase.auth.signUp()` |
| Sign in | `supabase.auth.signInWithPassword()` |
| Sign out | `supabase.auth.signOut({ scope: "global" })` |
| Password reset email | `supabase.auth.resetPasswordForEmail()` |
| Update password | `supabase.auth.updateUser({ password, current_password })` |
| Update profile | `supabase.auth.updateUser({ data: { full_name } })` |
| Get session | `supabase.auth.getSession()` |
| Auth state listener | `supabase.auth.onAuthStateChange()` |

### Password reset redirect

The reset email redirects to `{origin}/auth/reset-password`. This URL must be added to the **Redirect URLs** allowlist in Supabase Dashboard → Authentication → URL Configuration.

### Supabase Auth settings to verify

| Setting | Required value |
|---------|---------------|
| Email confirmations | Enabled (recommended) |
| Redirect URLs | Include `https://yourdomain.com/auth/reset-password` |
| Minimum password length | 8 (matches app-side validation) |

## API Routes

### `DELETE /api/delete-account`

Permanently deletes a user's account and all their travel records.

**Authentication:** Bearer token in `Authorization` header.

**Request:**
```
DELETE /api/delete-account
Authorization: Bearer <supabase_access_token>
```

**Success response:**
```json
{ "success": true }
```

**Error responses:**

| Status | Body | Cause |
|--------|------|-------|
| 401 | `{ "error": "Unauthorized" }` | Missing, invalid, or expired token |
| 500 | `{ "error": "Failed to delete account. Please try again or contact support." }` | Supabase operation failed (details in server logs) |
| 500 | `{ "error": "Server misconfiguration: SUPABASE_SERVICE_ROLE_KEY is not set." }` | Missing environment variable |

**Server-side flow:**
1. Extract and validate the Bearer token using the anon key client
2. Check `SUPABASE_SERVICE_ROLE_KEY` is set
3. Use admin client to delete all rows in `travel_records` where `user_id = user.id`
4. Use admin client to call `auth.admin.deleteUser(user.id)`
5. Return `{ success: true }`

**Note:** Detailed error information is written to server logs (`console.error`) but is not returned to the client, to avoid leaking internal database details.

## Environment Variables

| Variable | Where used | Description |
|----------|-----------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Client + server | Your Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Client + server | Supabase anonymous/public key — safe to expose in the browser |
| `SUPABASE_SERVICE_ROLE_KEY` | Server only (`/api/delete-account`) | Supabase service role key — bypasses RLS. Never expose to the browser. |

The `NEXT_PUBLIC_` prefix makes a variable available in browser bundles. The anon key is intentionally public — it is safe because RLS controls what it can access. The service role key must never have the `NEXT_PUBLIC_` prefix.

## localStorage Keys

The app stores three values in the browser's localStorage:

| Key | Type | Description |
|-----|------|-------------|
| `routebook-theme` | `"sand" \| "ocean" \| "sunset"` | Active colour theme |
| `routebook-home-country` | `string` | User's home country |
| `routebook-country-coordinates-v1` | JSON object | Cached `{ country: { lat, lng } }` map |

These are cleared on account deletion.
